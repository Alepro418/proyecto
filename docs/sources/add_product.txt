Crear un diagrama de secuencia UML para el proceso "Agregar Nuevo Producto" en un sistema de inventario.

Actores y Objetos (Lifelines):

:Administrador (Actor)
:Sistema (La interfaz con la que interactúa el usuario)
:ControladorProducto (Maneja la lógica de negocio para los productos)
:BaseDeDatos (Representa el almacenamiento de datos)
Flujo de Eventos (Mensajes):

El :Administrador ya ha iniciado sesión en el sistema.
El :Administrador selecciona la opción "Gestionar Productos" y luego "Agregar nuevo producto" en el :Sistema.
El :Sistema muestra un formulario para ingresar los datos del nuevo producto (ej. nombre, descripción, precio, stock inicial, proveedor).
El :Administrador completa el formulario con los detalles del producto y confirma la creación.
El :Sistema envía una solicitud para crear el nuevo producto al :ControladorProducto, pasando todos los datos ingresados.
El :ControladorProducto valida los datos recibidos (ej. que los campos requeridos no estén vacíos, que el formato sea correcto).
El :ControladorProducto ejecuta una operación de inserción (INSERT) en la :BaseDeDatos para guardar el nuevo producto.
La :BaseDeDatos confirma que el registro fue creado exitosamente.
El :ControladorProducto notifica al :Sistema que el producto ha sido agregado.
Finalmente, el :Sistema muestra un mensaje de confirmación al :Administrador (ej. "Producto agregado correctamente").

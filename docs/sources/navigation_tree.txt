Desarrollar un sistema de registro de inventario con la siguiente estructura de navegación y funcionalidades clave.

Estructura Central: Sistema de registro de inventario

Módulos y Funcionalidades Requeridas:

1.  Gestionar productos
    -   1.1. Agregar nuevo producto: Creación de nuevos ítems de inventario.
    -   1.2. Consultar lista de producto: Visualización de todo el inventario.
    -   1.3. Abastecer producto (sub-flujo de 1.2): Actualización o incremento del stock de un producto existente.
2.  Gestionar proveedores
    -   2.1. Registrar nuevo proveedor: Ingreso de datos de nuevos suministradores.
    -   2.2. Consultar lista de proveedores: Visualización del listado completo de proveedores.
    -   2.3. Consultar detalles del proveedor (sub-flujo de 2.2): Ver información detallada de un proveedor específico.
3.  Gestionar venta
    -   3.1. Registrar nueva venta: Creación de una nueva transacción de salida.
    -   3.2. Consultar historial de ventas: Acceso al registro cronológico de ventas.
    -   3.3. Ver detalles de una venta (sub-flujo de 3.2): Revisar el contenido y datos de una venta específica.
4.  Gestionar compra
    -   4.1. Registrar nueva compra: Creación de una nueva transacción de entrada (adquisición de inventario).
    -   4.2. Consultar historial de compras: Acceso al registro cronológico de compras.
    -   4.3. Ver tipo de compra: "Clasificación de la compra (ej: contado, crédito)."
5.  Autenticación
    -   5.1. Iniciar sesión: Proceso de login para acceso al sistema.
    -   5.2. Cerrar sesión: Proceso de logout para salir del sistema.
6.  Gestionar reportes
    -   6.1. Generar reportes del día: Informes resumidos de las operaciones diarias.
    -   6.2. Generar reportes de la semana: Informes resumidos de las operaciones semanales.
    -   6.3. Generar reportes del mes: Informes resumidos de las operaciones mensuales.
